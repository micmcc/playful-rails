# Playful Rails

A lightweight template for making games with intention.

This is not about writing mountains of code or heavy documents.  
It is about learning to **direct intent**: figure out what would be fun, sharpen the idea, keep a short living description of what you are building, then build → play → learn → improve in short loops.

Perfect for learners who already enjoy vibe-coding with Grok and want to understand the valuable work that sits *above* the code.

## The Loop (keep it short and fun)

1. **Explore** — Brainstorm ideas (divergent)
2. **Sharpen** — Turn a fuzzy idea into a clear Concept Card
3. **Intent** — Write / update a short living INTENT.md (the "what")
4. **Build** — Make the smallest playable slice
5. **Play & Reflect** — Play it, notice what feels good or confusing, capture learning
6. Repeat

The agent skills in `.grok/skills/` help keep this process on rails without lots of ceremony.

## Quick Start

1. Click **Use this template** on GitHub to create a new game repo.
2. Open the project with Grok Build (or your preferred tool).
3. Start with the `brainstorm` skill or just talk about game ideas.
4. Keep `INTENT.md` short and up to date — it is the map.

## Files that matter

| File | Purpose |
|------|--------|
| `INTENT.md` | Living one-pager of what this game is trying to be |
| `IDEAS.md` | Brainstorm dump + Concept Cards |
| `CHANGELOG.md` | What we tried, what we learned, what is next |
| `AGENTS.md` | Project rules and the creative loop for the agent |
| `.grok/skills/` | Four focused helpers for the loop |

No heavy PRDs. No big process. Just enough structure so changes stay coherent and the real work (deciding what is worth building and how it should feel) stays visible.

## Skills

- **brainstorm** — Generate and explore game ideas
- **concept-sharpener** — Turn a rough idea into a tight Concept Card + tiny POC suggestion
- **intent-keeper** — Help maintain and update the short INTENT.md
- **playtest-reflector** — Turn play notes into clear next experiments

Have fun. Keep the loops short. The code is the easy part.
